FAZ 1: CLI ARACI (Oncelik: YUKSEK)
------------------------------------------------------------------------
Amac: "elyan" terminal komutunu olustur. Tum sistemi CLI'dan yonetebilir hale getir.

Adim 1.1: CLI Iskeleti
  - cli/ dizini olustur
  - cli/main.py: argparse/click tabanli ana CLI parser
  - "elyan" komutu icin setup.py veya pyproject.toml entry_points tanimla
  - Alt komut yapisi: elyan <subcommand> [options]

Adim 1.2: Sistem Yonetim Komutlari
  - elyan doctor: sistem kontrolu (Python versiyonu, bagimlilklar, LLM erisimi, dizin izinleri)
  - elyan doctor --fix: otomatik duzeltme (eksik dizinler olustur, izinler ayarla)
  - elyan health: saglik durumu (CPU, RAM, disk, LLM latency, DB boyutu)
  - elyan status: agent durumu (calisma suresi, aktif gorevler, kanal baglantilari)
  - elyan status --deep: detayli analiz (tool saglik, LLM maliyet, bellek kullanimi)
  - elyan update: pip ile guncelleme

Adim 1.3: Gateway Komutlari
  - elyan gateway start: ana servisi baslat (agent + API server + Telegram bot)
  - elyan gateway stop: servisi durdur (PID dosyasindan)
  - elyan gateway status: servis durumu
  - elyan gateway restart: yeniden baslat
  - elyan gateway logs: log takibi (tail -f benzeri)
  - PID dosyasi: ~/.elyan/gateway.pid

Adim 1.4: Yapilandirma Komutlari
  - elyan config: interaktif yapilandirma sihirbazi (inquirer benzeri)
  - elyan config get <path>: JSON path ile deger oku (orn: models.default.provider)
  - elyan config set <path> <value>: deger yaz
  - elyan config unset <path>: deger sil

Adim 1.5: Bellek Komutlari
  - elyan memory status: bellek durumu (dosya sayisi, toplam boyut, son guncelleme)
  - elyan memory index: bellek indeksini guncelle (semantic embeddings yenile)
  - elyan memory search <query>: bellekte arama

Adim 1.6: Mesaj Komutlari
  - elyan message send --channel <channel> --text <text>: mesaj gonder
  - stdin'den pipe destegi: echo "Merhaba" | elyan message send --channel telegram

Adim 1.7: Dashboard Komutu
  - elyan dashboard: web dashboard'u tarayicida ac (webbrowser.open)

DOGRULAMA:
  - "elyan doctor" komutu calisir, sorunlari tespit eder
  - "elyan gateway start/stop/status" calisir
  - "elyan config get/set" elyan.json'u dogru okur/yazar
  - "elyan memory search" sonuc dondurur
  - "elyan message send --channel telegram" mesaj gonderir

FAZ 2: GATEWAY MIMARISI VE KANAL ALTYAPISI (Oncelik: YUKSEK)
------------------------------------------------------------------------
Amac: Tum kanallari yonetecek merkezi gateway olustur. Telegram'i bu
      gateway uzerinden calistir. Yeni kanal ekleme altyapisini kur.

Adim 2.1: Gateway Servisi
  - core/gateway/ dizini olustur
  - core/gateway/server.py: aiohttp tabanli HTTP sunucu (port 18789)
  - core/gateway/router.py: gelen mesajlari kanala gore agent'a yonlendir
  - core/gateway/message.py: UnifiedMessage modeli (kanal-agnostik mesaj formati)
    {
      id, channel_type, channel_id, user_id, user_name,
      text, attachments, reply_to, timestamp, metadata
    }
  - core/gateway/response.py: UnifiedResponse modeli
    {
      text, attachments, buttons, format, channel_hints
    }

Adim 2.2: Kanal Adaptoru Arayuzu
  - core/gateway/adapters/base.py: BaseChannelAdapter soyut sinifi
    - connect(): kanala baglan
    - disconnect(): baglanti kes
    - send_message(response): mesaj gonder
    - on_message(callback): mesaj dinle
    - get_status(): baglanti durumu
    - get_capabilities(): kanal ozellikleri (buton, dosya, ses vs.)

Adim 2.3: Telegram Adaptoru
  - core/gateway/adapters/telegram.py: TelegramAdapter(BaseChannelAdapter)
  - Mevcut telegram_handler.py logigini adapter'a tasi
  - handlers/telegram_handler.py -> adapter cagrilari
  - Geriye uyumlu: mevcut Telegram islevseligi korunur

Adim 2.4: Kanal Yonetim CLI
  - elyan channels list: tum kanallari listele
  - elyan channels status: baglanti durumu
  - elyan channels add: interaktif kanal ekleme
  - elyan channels remove <channel_id>: kanal sil
  - elyan channels login <channel_id>: kimlik dogrulama
  - elyan channels logout <channel_id>: oturum kapat
  - Kanal bilgileri elyan.json channels[] dizisinde saklanir

Adim 2.5: Webhook Endpoint'leri
  - POST /api/message: harici sistemlerden mesaj al
  - POST /hook/<event_name>: ozel webhook tetikleyicileri
  - GET /api/channels: kanal listesi
  - GET /api/channels/<id>/status: kanal durumu
  - Webhook kimlik dogrulama (bearer token)

DOGRULAMA:
  - Gateway port 18789'da calisir
  - Telegram mevcut gibi calisir (adapter uzerinden)
  - POST /api/message ile mesaj gonderilebilir
  - elyan channels list Telegram'i gosterir

FAZ 5: CRON ZAMANLAYICI VE HEARTBEAT (Oncelik: YUKSEK)
------------------------------------------------------------------------
Amac: Cron expression destekli gorev zamanlayici ve heartbeat sistemi kur.

Adim 5.1: Cron Engine
  - core/scheduler/ dizini olustur
  - core/scheduler/cron_engine.py: CronEngine sinifi
    - APScheduler (AsyncIOScheduler) tabanli
    - Cron expression destegi ("47 6 * * *" formati)
    - Her cron isi bir prompt: agent.process(prompt) cagrisina doner
    - is_id, expression, prompt, enabled, last_run, next_run, channel (sonuc nereye gider)
    - Isler elyan.json cron[] dizisinde saklanir
  - core/scheduler/job_runner.py: CronJobRunner
    - Agent'i cagir, sonucu belirtilen kanala gonder
    - Hata durumunda retry (3 deneme, exponential backoff)
    - Calisma suresi ve sonuc logu

Adim 5.2: Heartbeat Sistemi
  - core/scheduler/heartbeat.py: HeartbeatManager
    - Periyodik uyanma (varsayilan 6 saat)
    - Uyandiginda: sistem sagligi kontrol, bekleyen gorevleri isle
    - elyan.json heartbeat{enabled, intervalMinutes} ayari
    - Gateway calisirken aktif, durdugunca pasif

Adim 5.3: Cron CLI Komutlari
  - elyan cron list: zamanlanan gorevleri listele
  - elyan cron status: cron motoru durumu
  - elyan cron add --expression "0 6 * * *" --prompt "Daily briefing" --channel telegram
  - elyan cron rm <job_id>: gorevi sil
  - elyan cron enable/disable <job_id>: etkinlestir/devre disi birak

Adim 5.4: Proaktif Brifing Entegrasyonu
  - Mevcut core/proactive/briefing.py logigini cron job'a donustur
  - Sabah brifing: cron expression ile zamanla
  - E-posta triyaji: gunluk 2 kez
  - Trending haber ozeti: aksam 18:00

Adim 5.5: Webhook Tetikleyicileri
  - core/scheduler/webhook_trigger.py: WebhookTrigger
  - Gmail Pub/Sub: yeni e-posta -> agent.process()
  - GitHub webhook: PR/issue -> agent.process()
  - Genel webhook: POST /hook/<event> -> agent.process()
  - Webhook kayitlari elyan.json webhooks[] dizisinde

DOGRULAMA:
  - elyan cron add ile gorev eklenir ve zamaninda calisir
  - Heartbeat periyodik olarak uyananir
  - Webhook POST istegi agent'i tetikler
  - Sabah brifing cron ile zamanlanmis ve calisir
  - elyan cron list gorevleri dogru gosterir
