Local runtime first. Hosted access only where it belongs.

Install

Get Elyan running locally.

Use the real install path. Elyan is designed to run on the user's machine first, with Ollama as the preferred local model lane.

1
Install the binary
brew install --formula https://raw.githubusercontent.com/emrek0ca/elyan/main/Formula/elyan.rb
2
Start the local runtime
elyan start --port 18789
3
Open the desktop shell
elyan desktop
Local vs hosted

The boundary is explicit.

Local

Private memory, files, browser state, desktop actions, and local tool execution.

Hosted

Accounts, subscriptions, token balance, usage ledger, notifications, billing state, and downloads.

Optional lanes

Optional search, optional MCP, cloud assistance, and narrow VPS-backed control-plane features stay optional. They do not become the default brain.

Capabilities

Calm, direct, production-grade.
