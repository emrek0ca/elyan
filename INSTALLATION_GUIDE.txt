Step 2: Configure API Keys
────────────────────────────────────────────
Choose one or more LLM providers:

A. Groq (Free, Fastest) - RECOMMENDED
   1. Visit: https://console.groq.com
   2. Sign up (free)
   3. Get API key
   4. Run: export GROQ_API_KEY='gsk_...'

B. Google Gemini (Free)
   1. Visit: https://makersuite.google.com/app/apikey
   2. Create API key
   3. Run: export GOOGLE_API_KEY='AIza...'

C. Claude/Anthropic
   1. Visit: https://console.anthropic.com
   2. Create API key
   3. Run: export ANTHROPIC_API_KEY='sk-ant-...'

D. OpenAI (GPT-4)
   1. Visit: https://platform.openai.com/api-keys
   2. Create API key
   3. Run: export OPENAI_API_KEY='sk-...'


Persistent Setup (Recommended):
Create ~/.elyan/.env file:

cat > ~/.elyan/.env << 'EOF'
GROQ_API_KEY=gsk_...
GOOGLE_API_KEY=AIza...
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
EOF

Then add to ~/.bashrc or ~/.zshrc:
source ~/.elyan/.env


Step 3: Activate Environment
────────────────────────────────────────────

Option B: Python Interactive
────────────────────────────────────────────

from core.learning_engine import LearningEngine
from core.autonomous_coding_agent import AutonomousCodingAgent

# Initialize learning engine
engine = LearningEngine("user_001")

# Initialize coding agent
coder = AutonomousCodingAgent()

# Generate code
code = coder.generate_code("prime number checker")
print(code)

# Record interaction for learning
engine.record_interaction(
    "autonomous_coding",
    {"task": "prime checker"},
    code,
    success=True,
    duration=0.5
)

# Get recommendations
recommendations = engine.get_recommendations()
print(recommendations)
EOF

================================================================================
                        TROUBLESHOOTING
================================================================================

Problem: "Command not found: Python 3.9+"
Solution:
  1. Check version: python3 --version
  2. Install from: https://www.python.org/downloads/
  3. Or use: brew install python@3.11 (macOS)
  4. Or use: apt-get install python3.11 (Ubuntu)


Problem: "Virtual environment not found"
Solution:
  bash setup_elyan.sh --clean


Problem: "Git not found"
Solution:
  macOS: brew install git
  Ubuntu: apt-get install git
  Windows: https://git-scm.com/download/win


Problem: "API key not recognized"
Solution:
  # Verify it's set
  echo $GROQ_API_KEY

  # Set it
  export GROQ_API_KEY='gsk_...'

  # Or add to ~/.elyan/.env and source it


Problem: "Tests failing"
Solution:
  1. Check API keys are set
  2. Check internet connection
  3. Run: pip install -r requirements.txt
  4. Run: python3 -m pytest tests/test_weeks1112_modules.py -v


Problem: "Permission denied on /usr/local/bin/elyan"
Solution:
  sudo ln -s ~/.elyan/elyan /usr/local/bin/elyan

  Or just use full path:
  ~/.elyan/elyan --help


Problem: "Out of disk space"
Solution:
  # Clear cache
  rm -rf ~/.elyan/data/cache/*

  # Check disk usage
  du -sh ~/.elyan/

================================================================================
                        PERFORMANCE TIPS
================================================================================

For Speed:
- Use Groq provider (fastest)
- Reduce max_tokens to 2000
- Enable caching
- Use lower temperature (0.5)

For Accuracy:
- Use Claude or GPT-4
- Increase max_tokens to 4000
- Disable streaming
- Use higher temperature (0.8)

For Cost:
- Use Gemini (best value)
- Use smaller models
- Enable token optimization
- Batch requests

For Learning:
- Enable episodic memory
- Enable semantic knowledge base
- Record all interactions
- Review recommendations regularly


================================================================================
                        SECURITY BEST PRACTICES
================================================================================

1. NEVER commit API keys to git:
   Add ~/.elyan/.env to .gitignore

2. Use environment variables:
   export GROQ_API_KEY='...'
   # Not: hardcoded in config

3. Rotate API keys regularly:
   # Monthly for production

4. Monitor usage:
   tail -f ~/.elyan/logs/elyan.log

5. Enable audit logging:
   security:
     enable_audit_logging: true

6. Use HTTPS in production:
   security:
     require_https: true

7. Backup learning data:
   cp -r ~/.elyan/data ~/.elyan/data.backup

8. Update regularly:
   cd ~/.elyan/repo && git pull origin main
